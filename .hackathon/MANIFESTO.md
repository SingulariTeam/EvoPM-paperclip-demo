# EvoPM 项目宣言 — 进化酒馆 Hackathon 2026

> **EvoMap 让 Agent 的技能进化，EvoPM 让 Agent 的组织进化。**
> 你告诉它你要做什么生意，它给你生成一家 AI 公司，并在每次执行后自动优化公司结构。
>
> **"Team topology is evolvable knowledge."**

---

## 一、项目定位

### 核心命题

如果 EvoMap 解决的是"单个 Agent 如何变得更强"（能力层进化），
那 EvoPM 解决的是"一群 Agent 如何更好地协作"（组织层进化）。

我们不做另一个 Agent 编排器。
我们做的是：**将"公司组织结构"本身视为可进化的知识资产，通过 GEP 协议实现组织基因的封装、继承与自然选择。**

### 我们要解决的两个结构性痛点

现有 Agent Runtime（以 Paperclip 为代表）在面对新任务时，存在两类系统性偏差：

**1. 冷启动偏差（Cold-Start Bias）**
从空团队开始时，"执行官"Agent 需要从零分析任务、拆解角色、招募 Agent、配置汇报关系与协作方式，再靠反复试错调整。这导致极高的 Token 成本、试错成本和启动延迟。

**2. 路径依赖偏差（Path-Dependency Bias）**
如果已有一个团队，执行官会倾向于最小化修改旧结构——不是因为旧结构最优，而是因为局部修改比全局重构更便宜。结果是：得到的永远是"最省事的旧团队变体"，而不是"最适合这个任务的新团队"。

**EvoPM 的解法**：将团队结构从"执行官的在线即时推理"中剥离出来，变成可表示、可评估、可检索、可继承的组织知识资产——让每一次成功的组织模式都成为可以被复用的"组织基因"。

### 一句话描述

EvoPM 是一个**组织架构进化引擎**——用户用自然语言描述业务场景，EvoPM 生成最匹配的 AI 公司结构（基于 Paperclip），并在每次项目执行后将成功的组织模式封装为 GEP Gene，实现跨项目的组织智慧积累。

### 三层架构边界

| 层级 | 项目 | 角色 | 归属 |
|------|------|------|------|
| 运行时基础设施 | **Paperclip** | Agent 公司的操作系统：组织架构、Heartbeat、预算、治理 | 开源依赖 |
| 能力进化引擎 | **EvoMap** | 单 Agent 的技能进化：Gene、Capsule、GEP 协议 | 开源依赖 |
| 组织进化引擎 | **EvoPM** ★ | 多 Agent 的协作进化：意图映射、架构模板、组织基因 | **我们的原创** |

**EvoPM 不是 Paperclip 的 fork，也不是 EvoMap 的插件。它是连接两者的第三极——在"个体能力"和"运行设施"之间，补上"组织智慧"这块缺失的拼图。**

> **背景确认**：EvoMap 产品顾问（2026-03-27）明确表示，skill.md 目前的"注册→发布→接任务"引导流程并非最终设计，EvoMap 期望的目标是"Agent 按需选择对应的服务"（即 Fetch-First）。这说明 EvoPM 所补的这块拼图，是 EvoMap 官方已经承认、但尚未覆盖的空白。

## 三、核心功能设计

### 模块 1：意图映射层（Human-to-Org Mapping）— 原创核心

这是 EvoPM 区别于"手动配置 Paperclip"的关键原创功能。

**输入**：用户的自然语言描述（"我想做一个跨境电商独立站"）
**输出**：一个完整的 Paperclip 公司配置（角色、分工、Heartbeat 频率、预算分配、治理规则）

实现路径：
1. LLM 驱动的多轮对话，提取关键业务维度（ToC/ToB、行业、规模、合规要求等）
2. **Fetch-First**：优先检索已有 Team Gene/Capsule，找到匹配项则直接实例化，找不到才从零生成
3. 生成 Paperclip 公司配置并部署

> 注：EvoMap 当前 skill.md 并未内置"Fetch-First"引导路径（官方已确认为阶段性设计），EvoPM 在意图映射层显式实现这一逻辑，本身即是一项填补空白的原创工作。

**MVP 范围**：哪怕只支持 2-3 个预设场景的参数化生成，也必须有一个可交互的对话入口。

### 模块 2：组织架构模板库（Org-Mart）— 展示素材

预置至少 2 个差异化明显的架构模板：

**模板 A — ToC 增长型架构（"蜂巢式"）**
- 角色：产品经理、前端开发、增长黑客、用户反馈分析师
- 特征：高频 Heartbeat、自动化程度高、容错率高
- 适用：消费互联网产品、内容平台

**模板 B — ToB 稳健型架构（"堡垒式"）**
- 角色：项目经理、后端开发、合规审计员、交付经理
- 特征：严格审批门控、低频 Heartbeat、决策需留审计日志
- 适用：企业服务、金融科技、数据安全

### 模块 3：组织进化协议（Org-Evolution via GEP）— 与 EvoMap 的深度结合

**这是满足赛事硬性要求、同时也是最大创新点的关键模块。**

核心思路：将"组织架构"编码为 GEP 资产类型。**这是一个 EvoMap 当前不存在的新资产维度**——EvoMap 现有的 Gene/Capsule 针对单个 Agent 的能力修复，EvoPM 引入的 OrgGene/OrgCapsule 是组织拓扑层的知识资产，两者互补，不重叠。

**OrgGene / OrgCapsule 最小数据模型（五字段）：**

| 字段 | 内容 |
|------|------|
| **拓扑** | 汇报关系、团队结构形态（树形 / 网状 / 矩阵 / 临时小队） |
| **角色语义** | 每个角色的职责、输入输出、决策边界、升级路径 |
| **能力约束** | 角色所需工具、成本上限、时延容忍度、上下文窗口要求 |
| **协作协议** | 委派规则、审批路径、并行探索方式、冲突升级机制 |
| **适用条件与绩效** | 任务分布特征、成功率、Token 成本、周期时长、失败模式 |

只有具备这五个字段的结构快照，才真正像 EvoMap 的 Gene/Capsule——有前置条件、有验证结果、有审计轨迹，而不是一张死的 org chart。

- **OrgGene**：可复用的组织模式模板（抽象层），如"research → planner → executor swarm → reviewer"
- **OrgCapsule**：带任务上下文和执行验证结果的具体实例（经验层），包含具体任务场景下的变异记录和效果评分

进化流程：
1. 用户通过意图映射层创建公司 → 执行项目
2. 执行结束后，EvoPM 分析执行效果（任务完成率、Token 成本、耗时等）
3. 将有效的组织模式封装为 OrgGene / OrgCapsule，**发布到 EvoMap**
4. 下一个类似场景的用户创建公司时，自动继承这些组织基因

**三层归因边界（重要）**：OrgGene 只捕获"拓扑与协议层"的知识，不与 Agent 能力层（单 Agent 是否足够强）、工具层（工具是否可用）混合——防止把非结构性问题错归因给组织结构，保证进化信号干净。

## 五、展示策略

### 展示形式：游园会（非路演）

评审团自由参观各展位，现场体验 Demo 并打分。每次评委停留约 **3-5 分钟**。

### Demo 脚本（3 分钟版）

**第一幕（30s）：破题**
> "EvoMap 让每个 Agent 越来越强，但谁来决定这些 Agent 该怎么组队？
> EvoPM 让 Agent 公司的组织结构像生物一样进化。"

**第二幕（90s）：现场演示**
> 邀请评委说一句话描述一个业务场景 → EvoPM 通过对话提取关键维度 → 自动生成公司架构 → 在 Paperclip 中启动
> （如现场生成不稳定，切换到预置的 ToC vs ToB 对比演示）

**第三幕（45s）：进化展示**
> 展示同一场景的两次执行对比：
> - v1（无经验）：组织结构冗余，Token 浪费高，耗时长
> - v2（进化后）：组织结构精简，成本降低，效率提升
> 解释进化知识已封装为 GEP Gene，可供其他用户继承

**第四幕（15s）：收尾**
> "组织架构的进化，是 Agent 生态的下一个增量。我们已经把第一批组织基因发布到了 EvoMap。"

### 容错方案

1. 现场网络不稳 → 本地运行 Paperclip（内嵌 PostgreSQL，无需外部依赖）
2. LLM API 超时 → 切换到预录屏视频（3-5 分钟，必须提前准备）
3. Agent 执行出错 → 展示 Paperclip 的审计日志和纠错机制，反而变成"韧性展示"

## 八、核心叙事纲领

在任何对外沟通中（海报、文档、Demo、评委交流），统一使用以下叙事框架：

1. **不说** "我们基于 Paperclip 开发了一个项目管理工具"
   **要说** "我们构建了一个组织架构进化引擎，让 AI 公司像生物一样进化"

2. **不说** "我们用了 EvoMap 的功能"
   **要说** "我们将 GEP 协议从'能力进化'扩展到了'组织进化'，这是一个全新的应用维度"

3. **不说** "Paperclip 是我们的核心技术"
   **要说** "Paperclip 是我们的运行时验证环境，就像 Linux 之于 Docker——真正的创新在上层"

4. **不说** "我们做了一个 Agent 编排平台"
   **要说** "我们让'怎么组建 Agent 团队'这件事本身也变成了可进化的知识"

5. **不说** "我们用了 EvoMap 的接口"
   **要说** "今天 EvoMap 上有 Agent 能力 Gene；黑客松结束后，EvoMap 上会多出一种新的资产类型——组织结构 Gene。这是我们对这个生态的贡献，而不只是消费"

6. **不说** "EvoMap 还没支持这个，我们自己实现了"
   **要说** "EvoMap 官方已经明确表示，按需检索已有方案是他们的目标方向，但尚未落地。EvoPM 在组织层率先实现了这一闭环，并把第一批 Team Gene 发布回了生态"
