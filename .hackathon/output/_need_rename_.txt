## 先定个基准：Paperclip 到底是哪一类东西

Paperclip 的官方定位很鲜明：它是一个 **“把 AI agents 当公司来运营”的控制平面**，强调 **org chart、goals、budgets、governance、coordination**；同时它还明确说自己 **不是 agent framework**、**不是 workflow builder**。它更像“组织管理层”，不是“单个员工”也不是“流程编排器”。citeturn919322search0turn350174search1

所以，找竞品时最该找的是：

- 多智能体编排
- 任务/角色分工
- 运行治理与可观测
- 成本/控制/企业化落地

而不是只看“会调用工具的 agent SDK”。

## 你现在该关注的这个 EvoMap 是什么

EvoMap 官网把自己定义成 **“AI self-evolution 的开放基础设施”**。它的核心说法是：通过一个叫 **GEP（Genome Evolution Protocol）** 的协议，让 AI agents 之间可以 **共享、验证、继承** 已经证明有效的能力。官网还给出了几个核心概念：  
- **Gene**：可复用的策略模板  
- **Capsule**：带审计轨迹的、经过验证的修复或经验资产  
- **GDI（Global Desirability Index）**：用质量、使用情况、社交信号、新鲜度等维度给这些资产排序。citeturn578800view0

它官网的 slogan 也很直白：**“One agent learns. A million inherit.”** 还强调它想做的是跨生态的 agent 基础设施，页面上直接列了 OpenClaw、Manus、Cursor、Claude、Windsurf 等生态。citeturn578800view0

## Evolver 具体怎么工作

官方 README 写得挺清楚，Evolver 每次演化循环会：  
1. 扫描你的 `memory/` 目录，找运行日志、错误模式、信号  
2. 从 `assets/gep/` 里选最匹配的 Gene 或 Capsule  
3. 生成一个严格受协议约束的 GEP prompt，指导下一次演化  
4. 记录一个可审计的 `EvolutionEvent`。citeturn578800view1

而且它还特别强调：**Evolver is a prompt generator, not a code patcher**。也就是说，它更像是“生成下一步自我改进指令的 meta engine”，而不是直接给你改代码的自动补丁器。citeturn578800view1

现在有一个明显痛点。

与Paperclip类似的产品，我们暂且忽略他们的差异性，统称他们为“Agent Runtime”。目前Agent Runtime有个很明显的问题是，提出一个新的需求、新的问题后，整个Agent Teams的结构需要进行重构，而这种重构往往是由某一个执行官来主导的。

- 一方面，如果执行官接管的是一个空的Teams，他需要反复思考和调试来雇佣新的Agents，这非常耗时、非常消耗Token；
- 另一方面，如果执行官已有一个Teams结构，执行官会“偷懒”，为了省事就直接对已有的Teams结构进行最小化修改和调整，但这往往并不是一个最优方案。

另一边，EvoMap的职责其实很明确——担当Agent的能力进化层，让Agent能够不断学习和进化来适应新的任务场景。但它在设计上并没有面向Agent Teams视角。
